# Open Source Issues Roadmap

This roadmap lists issue ideas from beginner to advanced to help contributors improve this project and build a cool, representable GitHub profile.

## Beginner (Good First Issues)

1. Fix typo and grammar inconsistencies in UI text.
2. Improve button labels for clarity (for example, "Next Step", "Generate README").
3. Add missing `alt` text for decorative and meaningful images.
4. Standardize spacing and text sizes in footer and social sections.
5. Replace any remaining outdated brand/social references in docs.
6. Add concise empty-state messages where inputs are optional.

## Intermediate

1. Add form validation helpers for social links and usernames.
2. Improve accessibility: focus states, contrast checks, and keyboard navigation.
3. Add loading and error states for async preview generation workflows.
4. Refactor repeated social-link rendering into a reusable component config.
5. Add unit tests for markdown generation utilities and helpers.
6. Improve mobile responsiveness for multi-step forms and preview pane.

## Advanced

1. Add preset templates (minimal, developer, creator, data-science profiles).
2. Build live markdown linting and warning suggestions in preview.
3. Add drag-and-drop section ordering for generated README sections.
4. Introduce i18n support for key UI texts and generated section headings.
5. Add E2E test coverage (critical flow: input -> generate -> copy/download).
6. Optimize performance (code splitting, bundle analysis, and render profiling).

## Expert / Maintainer-Level

1. Design a plugin system for custom README section generators.
2. Add serverless analytics for anonymous feature usage trends.
3. Implement collaborative "share config via URL" with schema versioning.
4. Introduce robust CI gates: lint, test, accessibility, and build checks.
5. Create contribution automations (issue templates with difficulty tags).

## Suggested Milestones

1. **Milestone 1 - Polish Core UI**
   - Beginner and low-risk intermediate issues.
2. **Milestone 2 - Reliability and Accessibility**
   - Validation, test coverage, accessibility, and UX states.
3. **Milestone 3 - Power Features**
   - Templates, section ordering, and advanced customization.

## Contribution Goal

Every completed issue should improve one of these outcomes:

- Better visual quality for generated profiles.
- Better contributor experience for open-source developers.
- Better reliability, performance, and accessibility of the tool.
